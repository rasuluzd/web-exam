Accessibility Testing Report

We conducted an accessibility test with Google Lighthouse website to check for compliance with W3C WAI standards. Inclusive web design is important in the making of websites so eveyrone can participate on the internet independent of ones disability or age. With running these tests can we make sure that everyone has equal content to our pages and information on campus. We started running the automated tests on our introduction page, implemented the recommended changes to every site and and then progressed to the next page

Index.html Test

Missing attributes

Two issues was brought up in the first image, images not having alt attributes and links not having discernible names. An alt attribute specifies an alternate text for an image, in the case of the image not displaying correctly. The second point was about the links not having discernible names, which means that the link text is not descriptive enough to tell the user where the link will take them. To address both concerns, we added alt attributes to all images, but left the alt attributes empty of those images that is purely decorative. To make the links more descriptive, we added aria-labels and titles. An aria-label is a descriptor to give meaning to elements that do not have functional meaning in itself. For example the arrow that scrolls down to the main content, has an aria-label that says "Scrolls down to main content". We also added titles to links which will show a tooltip while hovering over a link that take you to another page on this website, or if it would go to another website where it would go.


The second image also is similar to our last problem with non-discernible link names, where the buttons on our index page used to only say 'Read More'. Hence, we changed the button text to 'Read More About Topical Section' and 'Read More About Accessibility Testing' to make it more descriptive.

Color Contrast

There was also the issue of color contrast, contrast is the difference in light between two colors, and is important for those with visual impairments. Hence we made sure to choose dark text on light backgrounds and light text on backgrounds with dark colors or shades. Also we added text shadow to the text on the hero of the introduction page, even though it passed the test, to make it even more readable.

Additional Changes

Some changes we did to enhance accessibility that did not show up in the test, changing from static px values to relative em and rem vales, making a more responsive design and allows users to resize text. Adding an underline to links which changes color while hovering. We also inverted the colors of the buttons so it would be more apparent it is hovered over. Also adding a thicker font to the buttons to make it more readable as a suggestion from a Teaching Assistant. After applying all of these changes, did we make a final test of every site for mobile and desktop view
